## 1. Cursor — what changed

### Canvas — interactive dashboards as durable artifacts
**Feature:** Cursor can respond with interactive canvases containing first-party components (tables, boxes, diagrams, charts) and existing components (diffs, to-do lists). Canvases are durable artifacts that live in the Agents Window side panel.
**Source:** https://cursor.com/changelog/04-15-26 · [announcement](https://cursor.com/blog/canvas)
**Status:** Shipped in Cursor 3.1.
**Practical implication:** Replaces hand-rolled HTML + JS dashboards with first-party components. Canvases are side-panel-native rather than file-based.

### Cursor 3.1 — tiled Agents Window, voice input, search filters
**Feature:** Agents Window now supports tiled panes for parallel agents. Voice input uses batch STT for higher-quality speech-to-text. Branch selection in the empty-state launcher. Diff-to-file navigation jumps directly to the exact line. Include/exclude filters in "Search in Files."
**Source:** https://cursor.com/changelog/3-1
**Status:** GA.
**Practical implication:** Enables multiple long-running agents side-by-side in the same window. Smaller quality-of-life improvements on voice input, file search, and diff navigation.

### Cursor CLI — `/debug`, `/btw`, `/config`, `/statusline`
**Feature:** CLI gains `/debug` (generates hypotheses, adds log statements, makes a targeted fix), `/btw` (ask a side question without derailing the main task), `/config` (interactive settings), `/statusline` (custom status bar). Model picker works from anywhere; image paste from clipboard; reliable `Ctrl+Z` backgrounding.
**Source:** https://cursor.com/changelog/04-14-26
**Status:** GA.
**Practical implication:** Narrows the gap between Cursor CLI and the IDE chat experience. `/debug` is a more structured alternative to ad-hoc debugging prompts.

### Bugbot — learned rules + MCP support
**Feature:** Bugbot learns from reactions/replies to PR comments and auto-promotes high-signal rules. Resolution rate 78%. Bugbot can now use MCP servers during code review.
**Source:** https://cursor.com/changelog/04-08-26 (landed just before the window, still relevant)
**Status:** GA.
**Practical implication:** PR-level AI review becomes team-tunable without manual rule authoring. MCP-during-review means Bugbot can validate claims against live tools (databases, issue trackers, etc.).

## 2. Claude Code — what changed

### `/ultrareview` — parallel multi-agent cloud PR review
**Feature:** Runs code review in the cloud using parallel multi-agent analysis and critique. Invoke with no args to review the current branch, or `/ultrareview <PR URL>` for a specific PR.
**Source:** [Claude Code CHANGELOG](https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md), Apr 16.
**Status:** GA.
**Practical implication:** Higher-quality PR review than single-agent critique. Useful as a first-pass gate before human review.

### `/less-permission-prompts` skill — auto-allowlist generation
**Feature:** Scans transcripts for routine read-only Bash and MCP tool calls and proposes a prioritized allowlist for `.claude/settings.json`.
**Source:** Claude Code CHANGELOG, Apr 16.
**Status:** GA.
**Practical implication:** Cuts permission-prompt friction without giving blanket consent. Reduces the repeated "Allow?" overhead for common read-only tool calls.

### Claude Opus 4.7 with `xhigh` effort level
**Feature:** Opus 4.7 available in Claude Code with a new `xhigh` effort level between `high` and `max`. Auto mode for Max subscribers. Interactive effort slider via `/effort`.
**Source:** [Anthropic news](https://www.anthropic.com/news/claude-opus-4-7), Apr 16.
**Status:** GA.
**Practical implication:** Higher-effort tier for multi-step work at lower cost than `max`. Cursor users: Cursor's own model picker already lists Opus variants; no action required. Claude Code users: `/effort xhigh` for complex tasks.

### Native Claude Code binary (replaces bundled JS)
**Feature:** CLI spawns a per-platform native Claude Code binary instead of bundled JavaScript. Faster startup, smaller footprint, same features.
**Source:** Claude Code CHANGELOG, Apr 17.
**Status:** GA.
**Practical implication:** Free upgrade. Anyone running Claude Code benefits immediately.

### Other notable Claude Code improvements (Apr 17–20)
- Faster `/resume` on large sessions (up to 67% on 40MB+) and faster MCP startup with multiple stdio servers. *Implication:* MCP-heavy configs benefit from the startup speedup.
- `/reload-plugins` and background auto-update now auto-install missing plugin dependencies from known marketplaces. *Implication:* hands-off plugin maintenance.
- Sandbox security hardening around `rm`, `find -exec`, and exec wrappers (`env`, `sudo`, `watch`, etc.). *Implication:* safer defaults; no action needed.
- Fixed MCP concurrent-call timeout handling where one tool call's message could silently disarm another call's watchdog. *Implication:* more reliable parallel MCP usage.
